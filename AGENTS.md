# 운영 명령어

- 의존성 설치는 `npm install`을 사용한다. 이 저장소는 `package-lock.json`을 사용하므로 yarn, pnpm, bun으로 바꾸지 않는다.
- 로컬 개발 서버: `npm run dev`.
- 프로덕션 빌드: `npm run build`.
- 린트: `npm run lint`.
- 변경 파일 또는 전체 저장소 포맷: `npm run format`.
- 쓰기 없이 포맷 확인: `npm run format:check`.
- 아직 테스트 스크립트는 없다. 로직을 바꿀 때는 테스트 프레임워크를 추가하기 전에 가장 작은 실행 가능한 검증을 남긴다.

# 골든 룰

## 불변 규칙

- 실제 패키지 경계가 생기기 전까지는 단일 Vite React 앱으로 유지한다.
- 비밀값, 생성된 `dist/`, 로컬 환경 파일을 커밋하지 않는다.
- React, TypeScript, 브라우저 API, CSS, 기존 도구로 처리 가능한 동작에는 의존성을 추가하지 않는다.
- 백엔드나 영속성 요구가 명시되기 전까지 UI 상태는 클라이언트에 둔다.

## 해야 할 것

- `package.json`의 기존 npm 스크립트를 재사용한다. 반복 명령이 실제로 생길 때만 스크립트를 추가한다.
- 추측성 기능 폴더보다 `src/` 아래에 작은 React 컴포넌트를 함께 둔다.
- 컴포넌트 경계와 브라우저 이벤트 경계에는 TypeScript 타입을 사용한다.
- 런타임 동작을 바꾸는 코드를 넘기기 전에 `npm run lint`와 `npm run build`를 실행한다.

## 하지 말 것

- 프로젝트 설정 안내를 여기에 중복하지 않는다. 사람을 위한 설정 문서가 필요하면 `README.md`를 업데이트한다.
- 라우팅, 전역 상태, CSS 프레임워크, API 클라이언트, 테스트 프레임워크를 선제적으로 도입하지 않는다.
- ESLint나 TypeScript 오류를 넓은 disable로 우회하지 않는다. 원인을 고치거나 필요한 한 줄만 좁게 격리한다.

# 프로젝트 컨텍스트

Cabin Monitoring은 현재 모니터링 경험을 위한 최소 브라우저 UI 스캐폴드다.

기술 스택: Vite, React 19, TypeScript 6, ESLint 10, Prettier 3, npm.

# 표준 및 참조

- 소스 파일은 ES 모듈, React 함수 컴포넌트, `src/app.tsx`에 이미 쓰인 소문자 앱 엔트리 명명을 따른다.
- 포맷은 Prettier가 담당하고 린트 규칙은 `eslint.config.js`에 있다.
- Git 커밋은 Conventional Commits를 사용하며, 기존 워크플로와 맞을 때는 한국어 설명을 우선한다.
- 유지보수 정책: 이 규칙이 코드나 스크립트와 어긋나면 같은 변경에서 `AGENTS.md`를 업데이트하거나 불일치를 명시한다.
- 에이전트가 어떤 스킬을 사용하든 사용자에게 출력하는 설명, 진행 상황, 최종 응답은 한국어로 작성한다. 코드, 명령어, 로그, 식별자는 원문을 유지한다.
